# Plan 01-01: Technical SEO Rescue & Canonical Cleanup

## Goal
Eliminate search engine redirects and indexing errors by aligning all navigation paths to extension-less clean URLs and setting strict canonical routing.

## Requirements
- SEO-01: Fix Google Search Console "Page with redirect" and "Redirect error" mapping.
- SEO-02: Ensure canonical tags correctly match the actual deployment URLs to resolve indexation confusion.
- SEO-03: Remove `.html` extensions from internal linking (navigation menus, footer, and sitemap.xml) to match server cleanly.

## Tasks

<task>
<action>
Remove all `.html` extensions from internal `href` attributes across all HTML pages (`index.html`, `about.html`, `location.html`, `amenities.html`, `contact.html`, `price-details.html`, `plot-sizes.html`, `gallery.html`, and `blog/*.html`). 
For example, change `<a href="about.html">` to `<a href="/about">` (or `<a href="about">` depending on current relative paths, ensuring it resolves cleanly). Note: Ensure index.html resolves simply to `/`.
</action>
<read_first>
- `index.html`
- `about.html`
- `blog/index.html`
</read_first>
<acceptance_criteria>
- Internal link href attributes within the site do not contain `.html`.
</acceptance_criteria>
</task>

<task>
<action>
Add or update the `<link rel="canonical" href="https://omaxeeternity2.in/..." />` tag within the `<head>` of every HTML page to strictly reflect the clean URL route. 
For `index.html`, it should be `href="https://omaxeeternity2.in/"`. 
For `about.html`, it should be `href="https://omaxeeternity2.in/about"`.
</action>
<read_first>
- `index.html`
- `about.html`
</read_first>
<acceptance_criteria>
- `<link rel="canonical"` points to the strict HTTPS non-.html naked domain path.
</acceptance_criteria>
</task>

<task>
<action>
Update `sitemap.xml` to replace all `*.html` `<loc>` URL references with the final clean naked URL paths.
For example, `<loc>https://omaxeeternity2.in/about.html</loc>` becomes `<loc>https://omaxeeternity2.in/about</loc>`.
</action>
<read_first>
- `sitemap.xml`
</read_first>
<acceptance_criteria>
- `sitemap.xml` does not contain `.html` substrings inside `<loc>` nodes.
</acceptance_criteria>
</task>

<task>
<action>
Review `script.js` to strip out any hardcoded `.html` path usages in `window.location.href` assignments or routing logic, matching the updated link paths.
</action>
<read_first>
- `script.js`
</read_first>
<acceptance_criteria>
- `.html` strings are purged from navigation actions in the JS file.
</acceptance_criteria>
</task>

## Verification

### Must Haves
1. Internal HTML links seamlessly reference extension-less URLs.
2. The `sitemap.xml` reflects clean route representations.
3. Canonical links exclusively present the canonical naked URL to web crawlers.

### Verification Steps
1. Execute string match over codebase: `grep -E "href=\".*\.html\"" *.html`. Expect 0 results except for explicitly external links.
2. Read `sitemap.xml` and guarantee `<loc>` URLs do not terminate in `.html`.
3. Read `index.html`, `contact.html`, and `location.html` to guarantee their `<link rel="canonical">` matches their clean `https://omaxeeternity2.in/...` endpoint.
